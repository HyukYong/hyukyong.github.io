About Qemica

Computational chemistry, built for Quantum.

Qemica is a computational-chemistry workbench for the researchers who actually run the calculations — rigorous about the science it does today, and built on an architecture ready for the quantum era.

Why we build it

Most chemistry tooling makes you trade power for clarity — string together a builder, an input file, a parser, and a plotter, and lose the thread of the science in between. Qemica keeps the molecule, the project, and the design language constant from first sketch to final figure.

We build for working chemists: open formats you can read with any tool, numbers in honest units, and methods labeled by where they actually run. Nothing is hidden, and nothing is oversold.

Our rule: never sell a qubit doing work it can't yet do. Classical by default, on an open-source stack — hardware qubits opt-in, labeled by what actually runs there.

What we stand for

A few principles, applied everywhere.

01

Honest by default

We label every method by where it runs — classical engine, simulator, or QPU — and would rather under-promise the qubits than oversell them.

02

Open & interoperable

Projects are plain folders, energies in honest units, built on an open-source stack — PySCF, Qiskit, OpenFermion — you can inspect and trust.

03

Quantum-ready

A complete classical workflow today, behind an interface that won't change when fault-tolerant quantum methods become practical.

04

For working chemists

Built for the people who actually run the calculations — sensible defaults for every job, full control when you need it.
